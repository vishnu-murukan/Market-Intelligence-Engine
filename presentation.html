Technical Implementation

Key design decisions

🌐 Google Search Grounding

Every data-fetching call uses Gemini's built-in web search — no scraping, no third-party APIs. Data is live, not from training cutoff.

📡 SSE Streaming

Results stream to the frontend as each stage completes. Users see progress in real-time — no waiting for a full 2-minute pipeline to finish.

🔄 Parallel Execution

Stages 2-4 (events, contacts, activity) run in parallel via ThreadPoolExecutor — cutting total pipeline time by ~40%.

🛡️ Graceful Fallbacks

If live API fails: cache → demo data → empty state. Rate limit auto-retry with exponential backoff. Never crashes.

💾 Smart Caching

Results cached for 6 hours. Cache only accepted if ≥10 brands — stale caches with fewer entries are automatically busted.

📤 Export

Full results exportable as JSON, CSV (prospects table), or TXT (formatted report) — ready for CRM import or team sharing.

Data Integrity

No fabrication — public data only

  • 🌐Google Search Grounding — every call searches the live web, not training data
  • 🔗Source attribution — every data point has a source_url you can click and verify
  • 🛡️Confidence labels — HIGH / MEDIUM / LOW transparency on data quality
  • 🌡️Low temperature (0.2) — factual accuracy prioritized over creativity for data calls
  • ⚠️"Not publicly listed" — contacts say this explicitly instead of making up emails
  • 🏭Works on ANY industry — no hardcoded data, no reconfiguration needed
Verification approach

Every piece of data is verifiable:

• Brand homepages → clickable source_url
• Event pages → official event URLs
• Contact profiles → LinkedIn profile links
• Press emails → from company /press pages

The judges can click any link in the demo to verify the data is real.
